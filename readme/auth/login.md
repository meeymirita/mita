====================
ЛОГИН ПОЛЬЗОВАТЕЛЯ
====================

Метод: POST
Эндпоинт: /api/user/login


В форме name="login" или FormData ключ login другое не пройдёт 
Зайти можно по email\login

Тело запроса (JSON):
{
"login": "mirita1@gmail.com",
"password": "qweqweqweqw",
"password_confirmation": "qweqweqweqw"
}

1. Email\Login:
    - Обязательное поле
    - Должен быть корректным email адресом
    - Должен быть уникальным (не зарегистрирован ранее)
    - Максимум 255 символов

2. Password:
    - Обязательное поле
    - Минимум 3 символа
    - Должен совпадать с password_confirmation
    - Должен содержать:
        * Хотя бы одну заглавную букву
        * Хотя бы одну строчную букву
        * Хотя бы одну цифру
