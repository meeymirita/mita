====================
РЕГИСТРАЦИЯ ПОЛЬЗОВАТЕЛЯ
====================

Метод: POST
Эндпоинт: /api/user/register

Тело запроса (JSON):
{
"email": "mirita@gmail.com",
"password": "mirita123QWE",
"password_confirmation": "mirita123QWE"
}

Правила валидации:

1. Email:
    - Обязательное поле
    - Должен быть корректным email адресом
    - Должен быть уникальным (не зарегистрирован ранее)
    - Максимум 255 символов

2. Password:
    - Обязательное поле
    - Минимум 3 символа
    - Должен совпадать с password_confirmation
    - Должен содержать:
        * Хотя бы одну заглавную букву
        * Хотя бы одну строчную букву
        * Хотя бы одну цифру

Сообщения об ошибках:
- "Поле email обязательно для заполнения"
- "Введите корректный email адрес"
- "Этот email уже зарегистрирован"
- "Email не должен превышать 255 символов"
- "Поле пароля обязательно для заполнения"
- "Пароль должен содержать минимум 3 символа"
- "Пароли не совпадают"
- "Пароль должен содержать хотя бы одну заглавную букву, одну строчную букву и одну цифру"

После успешной регистрации:
1. На указанный email отправляется письмо с 6-значным кодом подтверждения
2. Пользователь должен ввести этот код для подтверждения email

Варианты интерфейса:
1. После окна регистрации сразу показывать поле для ввода кода
2. Отдельная страница/окно для ввода кода

====================
ПОДТВЕРЖДЕНИЕ EMAIL
====================

Метод: POST
Эндпоинт: /api/user/verify-email

Тело запроса (JSON):
{
"email": "mirita@gmail.com",
"code": "131008"
}

Процесс:
1. Пользователь получает email с кодом (пример: 131008)
2. Вводит email и код в форму / или просто код email сохранить в переменную можно
3. Если код верный - email подтверждается
